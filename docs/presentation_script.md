# 项目讲解稿

## 1. 开场与背景
- 简要自我介绍后说明研究背景：云南主要农作物产量受气候波动、耕作制度与市场变化等因素影响，仅凭经验难以支撑精细化决策，因此需要数据驱动的预测与可视化系统。【F:docs/thesis_excerpt.txt†L1-L50】
- 点出论文与项目定位：基于 Spring Boot + Vue3 的前后端分离架构，实现数据管理、预测建模与结果展示的完整闭环。【F:README.md†L1-L36】【F:docs/thesis_excerpt.txt†L32-L49】

## 2. 系统架构与技术选型
- 架构总览：后端 Spring Boot 3.5 + MySQL + Spring Security + JWT；前端 Vue3 + Vite7 + Element Plus + ECharts，前后端通过 REST 接口对接。【F:README.md†L5-L66】
- 目录说明：`demo/` 为后端（控制器、服务、实体、安全配置）；`forecast/` 为前端（组件、布局、业务视图、Pinia 状态）。【F:README.md†L7-L26】

## 3. 业务功能亮点
- 鉴权与账号：验证码、注册、登录、刷新令牌、当前用户查询与登录日志记录。【F:README.md†L41-L55】
- 基础资料：作物与行政区域的增删改查。【F:README.md†L41-L50】
- 数据导入与管理：登记原始文件、录入产量/价格、上传 CSV/Excel、查看任务状态与明细、批量清理任务记录。【F:README.md†L41-L50】
- 预测管理：维护模型与任务、触发预测、查询历史记录，支持 Prophet/LSTM/ARIMA 等模型并持久化参数。【F:README.md†L47-L50】【F:docs/thesis_excerpt.txt†L32-L49】
- 报告与仪表盘：生成预测报告，聚合历史产量和区域对比，为驾驶舱图表提供数据。【F:README.md†L49-L51】
- 系统管理：系统参数、操作日志与管理员用户管理，基于 JWT 的精细化权限控制。【F:README.md†L50-L66】

## 4. 页面与演示流程
1) **登录/注册**：演示图形验证码、记住我、找回密码/邮箱验证码，验证登录后重定向与角色守卫。
2) **仪表盘**：展示导入概览数字与图表，切换作物/区域/年份筛选，查看历史预测表格并支持导出或删除记录。【F:README.md†L69-L96】
3) **数据中心**：上传 CSV/Excel，观察导入任务进度与明细，清理历史任务，查看最新预览表格。【F:README.md†L43-L50】【F:README.md†L69-L96】
4) **预测中心**：配置地区、作物、模型、历史窗口与预测步数，调整气象与模型参数，触发预测并查看结果列表和执行建议。【F:README.md†L47-L50】【F:docs/thesis_excerpt.txt†L32-L49】
5) **预测可视化与报表**：选择预测记录，切换折线/柱状等图表，查看预测明细与历史对比，输出报告用于汇报。【F:README.md†L49-L51】【F:README.md†L69-L96】
6) **用户与系统管理**：分页查看用户、重置密码、分配角色；查看操作日志与系统参数，说明权限模型如何保护接口。【F:README.md†L50-L66】

## 5. 数据与模型设计
- 数据库：`schema.sql` 预置 20 余张表，覆盖作物、区域、数据集、预测运行、报表及权限体系；`data.sql` 写入基础作物、角色与管理员账号，确保开箱即用。【F:README.md†L56-L63】
- 预测模型：默认创建包括 Prophet 节假日回归在内的多种模型，支持节假日与季节性因素建模，兼顾趋势和鲁棒性。【F:docs/thesis_excerpt.txt†L32-L49】
- 评估与可视化：使用 ECharts 展示产量统计与预测结果，可对不同模型效果进行指标评价与对比。【F:docs/thesis_excerpt.txt†L32-L49】【F:README.md†L69-L96】

## 6. 演示节奏建议
- **启动准备**：`./mvnw spring-boot:run` 启动后端，`npm run dev` 启动前端，使用默认管理员 `admin/Admin@123` 登录。【F:README.md†L27-L40】【F:README.md†L76-L90】
- **核心路径演示**：登录→上传数据→配置模型并触发预测→查看仪表盘与预测可视化→导出报告。
- **补充亮点**：强调 JWT 权限、自动化测试覆盖注册/登录/刷新链路、可通过 `.env.local` 指向后端切换环境。【F:README.md†L41-L66】【F:README.md†L91-L104】

## 7. 总结
- 强调系统实现了数据管理、预测建模、可视化决策的闭环，降低重复操作成本，为智慧农业场景提供可落地的工程方案。【F:docs/thesis_excerpt.txt†L32-L50】
- 可进一步扩展模型库、完善自动化测试与部署管道，以适应更大规模的生产使用。【F:README.md†L91-L104】
